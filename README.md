# 良いコード/悪いコードで学ぶ設計入門 - React/Next.js 版

『良いコード/悪いコードで学ぶ設計入門』の設計原則を React/Next.js で実践的に学ぶプロジェクトです。

## 📚 学習内容（全18章）

### 第1章: 悪しき構造の弊害を知覚する
- 意味不明な命名
- 条件分岐のネスト
- データクラスの問題点

### 第2章: 設計の初歩
- 意図が伝わる命名
- 変数の使い回しを避ける
- 関係するデータとロジックをクラスにまとめる

### 第3章: カプセル化の基礎
- 完全コンストラクタ
- 値オブジェクト
- 不変性の重要性

### 第4章: 不変の活用
- 再代入の防止
- 副作用のデメリット
- 不変と可変の使い分け

### 第5章: データとロジックのカプセル化実践
- プリミティブ型執着
- staticメソッドの誤用
- 多すぎる引数

### 第6章: 関心の分離
- インスタンス変数ごとのクラス分割
- インターフェイスと実装の分離

### 第7章: 関心の混在を分離する実践
- 単一責任の原則
- 継承より委譲
- 巨大クラスの対処法

### 第8章: 条件分岐
- 早期returnでネスト解消
- interface/ストラテジパターン
- フラグ引数の回避

### 第9章: コレクション
- ループ処理中の条件分岐
- コレクション処理のカプセル化

### 第10章: 設計を損なう悪魔たち
- デッドコード
- マジックナンバー
- null問題
- グローバル変数

### 第11章: 名前設計
- 目的駆動名前設計
- 技術駆動命名の回避
- 意図がわかる名前

### 第12章: コメント
- 退化コメントの回避
- 目的や仕様変更時の注意点を伝える
- ドキュメントコメント

### 第13章: メソッド（関数）
- コマンド・クエリ分離
- 引数の扱い方
- 戻り値の設計

### 第14章: モデリング
- 目的別モデリング
- 単一責任と単一目的
- 深いモデル

### 第15章: リファクタリング
- ネストの解消
- テストコードを用いた安全なリファクタリング
- IDEのリファクタリング機能

### 第16章: 設計の意義
- 設計しないと生産性が低下する理由
- コード品質を判断する指標
- 設計対象と費用対効果

### 第17章: 開発プロセスとの戦い
- コミュニケーション
- 設計と実装のフィードバック
- コードレビュー
- チームの設計力向上

### 第18章: 設計技術の深め方
- 推薦図書紹介
- 設計スキルを高める学び方
- アウトプット重視の学習

## 🚀 セットアップ

```bash
# 依存関係のインストール
pnpm install

# 開発サーバーの起動
pnpm dev

# コードフォーマット
pnpm format
```

## 🛠 使用技術

- **Next.js 15** - App Router
- **React 19**
- **TypeScript**
- **Tailwind CSS 4**
- **shadcn/ui** - UIコンポーネント
- **Biome + Ultracite** - Linter/Formatter

## 📖 学習の進め方

1. トップページから各章にアクセス
2. 「悪い例」と「良い例」を比較
3. 実際にコードを書いて理解を深める
4. React/Next.jsの文脈で設計原則を理解する

## 💡 このプロジェクトの特徴

- 書籍の内容をReact/Next.jsの文脈に置き換え
- TypeScriptの型システムを活用した安全な設計
- Reactコンポーネントとフックの設計パターン
- 実際に動作するコード例で学習
